## Design Context

### Users
Security professionals — red teamers, security engineers, SOC analysts, and security leadership at mid-to-enterprise organizations. They use ProjectAchilles to validate defenses, measure detection coverage, and prove security ROI. Context: focused, technical, high-stakes decisions. They value precision and clarity over decoration.

### Brand Personality
**Precise. Trusted. Formidable.**
- Authoritative without being corporate
- Technical depth with design sophistication
- Evidence-driven — measurement over opinion

### Aesthetic Direction
- **Visual tone**: Modern premium — inspired by Linear/Vercel's restraint and fluidity, but with the gravitas of a security platform
- **Theme**: Dark-first, deep backgrounds (project uses `oklch` color system with hero palette: deep navy `#050810` to elevated `#0f1528`)
- **Accent**: ProjectAchilles green (`#00e68a` / `#00ffaa`) — confidence, readiness, "systems go"
- **Typography**: Orbitron (display), Rajdhani (UI labels), JetBrains Mono (code), system fonts (body)
- **Motion**: Purposeful — smooth reveals, subtle state transitions. No gratuitous animation.
- **Anti-references**: Generic SaaS login (centered form + logo + white bg), corporate blue dashboards, no personality
- **Emotional goal**: Confidence and security — you feel your data is protected, the tool is solid and trustworthy

### Design Principles
1. **Earn trust visually** — every element should reinforce that this is a security platform built by people who understand security
2. **Restrained power** — the design is capable of being loud but chooses precision. Details over spectacle.
3. **Intentional space** — generous whitespace, clear hierarchy, nothing competes for attention
4. **Fluid and alive** — subtle motion that makes the interface feel responsive and present, not animated for show
5. **Dark confidence** — deep backgrounds with controlled luminance, accent colors used sparingly for maximum impact
