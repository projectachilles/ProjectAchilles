Gotcha — table recreation
SQLite has no ALTER COLUMN. Three CHECK-constraint expansions live in database.ts: migrateDarwinConstraint, migrateExecuteCommandType, migrateUninstalledStatus. Pattern: FK OFF → DROP IF EXISTS temp (crash-recovery) → CREATE temp → INSERT SELECT → DROP old → RENAME → recreate indexes → FK ON. Always call database.pragma('foreign_keys = OFF') — never exec('PRAGMA ...') (PRAGMA only works outside transactions).

Elasticsearch — analytics + correlation

Two indices. Both target ES 8.x — client pinned to @elastic/elasticsearch 8.x because v9 sends compatible-with=9 headers that ES 8.x rejects with 400.

The composite test_uuid

Bundle controls fan out as independent ES documents but need to be groupable back to their parent. The composite key uses :: as separator — unambiguous because neither UUIDs nor control IDs ever contain colons.

parent bundle UUID
7659eeba-f315-440e-9882-4aa015d68b27
::
control ID
CH-IEP-003

Use uuid.split('::') to decompose. Standalone (non-bundle) tests use a plain UUID with no ::.
